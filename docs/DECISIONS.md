# 컴페이스 — 결정 기록 (Decision Log / ADR)

> **역할:** 각 결정의 **근거 아카이브(WHY)**. "왜 X를 택했나"를 조회하는 곳.
> **정본 경계:** 의도·북극성 = `VISION.md` · 현재 확정 스코프 = `SPEC.md` · 화면 로직 = `SCREEN-FLOW.md` · 불변 규칙 = `/CLAUDE.md`.
> 현재 상태 선언이 `SPEC.md`와 어긋나면 **SPEC이 최신**이다. 여기 항목은 "그 결정을 왜 내렸는가"만 담고, 최신 스코프는 각 항목의 `→ SPEC` 링크를 따른다.
> 변경 이력은 맨 아래 [Changelog](#changelog) 한 곳. 개별 항목 본문엔 버전 태그를 달지 않는다.

## D-01

**결정:** 일반 웰니스/생산성 앱(B2C)으로 출시. '치료' 클레임 없음.
**대안:** 디지털 치료기기(DTx) — 식약처 SaMD 인허가 + 임상.
**근거:** 빠른 출시·시장 테스트, 인허가 시간·비용 회피. "약물=엔진, 훈련=네비게이션" 시너지로 *약물 병행 보조*와 *증상 호전 후 평생 루틴*을 하나로 잡음.
**단서:** 이 시너지는 약물이 켜져 있을 때의 이야기. 약물 중단자(K)에겐 엔진이 없으므로 "약물 없이도 버티는 최소 버팀목"을 바닥으로 깐다([D-18](#d-18)). 마케팅 표현은 의료법 회피 가이드([부록A](#부록a-법적ip)) 준수.

## D-02

**결정:** 타겟 = 정해진 일과 없이 24시간을 스스로 기획하는 취업 준비생(페르소나 A).
**대안:** 외부 압박이 명확한 주니어 직장인 / 자녀 교정 부모.
**근거:** 외부 강제성의 부재가 ADHD에게 가장 치명적(직장인은 압박으로 벼락치기라도 함, A는 강제성 0 → 가장 취약). DTx 임상에서 성인 효과 크기가 아동의 약 7배. A는 **획득·마케팅용**이며 설계 검증은 K가 담당([D-03](#d-03)).

## D-03

**결정:** 제품의 바닥(floor)을 정의하는 정서적 극단 페르소나 K를 **설계 기준선**으로 삼는다. 원칙 = **"5%의 K에게 통하면, 모두에게 통한다."** (A를 대체하지 않음.)
**K 프로필:** 29세, ADHD 약물 중단·항우울제만 복용, 우울 동반, 심리 배터리 5%, 목표가 *의무(공무원)↔열망(웹소설)*으로 찢김.
**근거 — K가 무너뜨린 세 가정:** ① 엔진이 꺼졌다("15분"이 천장인 날) → [D-18](#d-18). ② 북극성이 둘로 찢겨 충돌 → [D-19](#d-19). ③ 고독한 보상이 결핍을 못 메우나, 외부 관객으로 채우면 K를 부순 구조 재생산 → [D-20](#d-20). + 가로지르는 위험: 반발심 → [D-21](#d-21).
**검증:** K는 상주·상시 인터뷰 가능한 실재 인물. §0 성공지표("덜 미워하며 잠들었는가")는 앱이 측정하지 않고 K 인터뷰로 앱 밖에서 정성 관측. **리스크:** n=1 근접 과적합 → `SPEC.md` 리스크 레지스터 R1.

## D-04

**결정:** 4대 페인포인트(시간 맹점·우선순위 마비·시동 걸기 지연·에너지 고갈) 중 **#3 시동 걸기 지연**을 최우선 기반으로 구축.
**근거:** 4대는 사슬 연결. 과제 쪼개기(Chunking)를 중심 뼈대로 두면 나머지가 자연 확장(시간 맹점→타이머 결합, 우선순위→One Task, 에너지→"오늘 1개만"). Solanto CBT "시작이 힘들면 단계가 큰 것"과 일치.

## D-05

**결정:** 우선순위 '판단'은 빼고, 쪼개진 것 중 **'가장 만만한 하나'**를 사용자가 직접 가볍게 고른다.
**대안:** 중요도/긴급도 판단을 사용자에게 요구(임상 표준) / 앱이 대신 선택.
**근거:** 우선순위 판단 자체가 #2 마비를 재발시킴. 앱이 대신 고르면 수동 원칙([D-07](#d-07))과 충돌, 중요도 줄세우기는 #2와 충돌. '만만함' 직감 축이 양쪽을 모두 피함. 우선순위 코칭은 후순위 킵.

## D-06

**결정:** 워크플로우 = UX/행동 흐름 우선(Task-Driven Design).
**대안:** 기능 우선.
**근거:** 기능 과잉(Feature Bloat)이 인지 부하를 가중. 화면(와이어)에서 필수 기능을 역산. 3·4단계 산출물은 별도 화면 설계도(`SCREEN-FLOW.md`)로 분리.

## D-07

**결정:** 과제 분할 = 수동 입력(Manual). AI 자동 분할 ❌.
**근거:** "잡는 법을 알려준다" 철학의 직접 구현. 직접 쪼개는 행위가 실행 기능을 자극하는 인지 훈련. 토큰 비용 회피.

## D-08

**결정:** 쪼개기 키워드 = 구체적/행동 동사(검색하기·읽기·타이핑하기)를 칩으로 제공, [대상]+[행동 동사] 조합.
**대안:** 추상적/단계별(준비·실행·점검).
**근거:** 백지 입력은 "무엇부터?" 과부하. 단계어는 매번 해석(번역) 비용 → 마찰. 구체 동사는 "지금 브라우저 켜서 검색하면 되는구나"의 즉각 실행성 확보. OST 작업기억 보조 + 마이크로 해빗과 일치.

## D-09

**결정:** 완료 기준 = 시간 기반(Time-Boxing). **15분 타이머를 채우면 결과물 퀄리티 무관 무조건 100% 완료.**
**대안:** 체크리스트 / 개수·물리 상한.
**근거:** 체크리스트는 무한 증식 → 시작 압박 + 완벽주의 결합. 개수 상한은 변수 많아 AI 의존·비용↑. 시간은 보편 불변 규격이라 가장 확실한 완료 정의이자 도파민 보상.
**주의:** "에너지 점등=시간기반(무조건)"과 "완료/미완료=이월 판단용 별개 라벨"의 관계는 `SCREEN-FLOW.md` P12에서 정리. → 해석: 둘 다 같은 에너지, 라벨만 이월용.

## D-10

**결정:** 진행도 = **오늘의 행동 에너지 바.** 15분짜리 칸이 하나씩 점등(빈 칸→채워진 칸). 라벨은 "N블록"(분 환산 금지).
**대안:** 전체 목표 달성도(%) / 매끄러운 연속 progress bar.
**근거:** 거대 목표까지 몇 %는 측정 불가·무의미. 블록=게임 경험치 단위. "45분"으로 환산하면 "겨우 45분?" 생산성 프레임으로 미끄러져 톤 위반. 한 칸=한 번의 시동=한 번의 승리. 점등은 **블록 종료 그 순간** 즉시(지연 최소화, [D-20](#d-20)).
**→ SPEC:** 미리 그린 빈 칸(할당량) 폐기, 0에서 자라남, 날짜 간 비교 그래프 금지 → `SPEC.md` 에너지 바 §.

## D-11

**결정:** 타이머 종료 후 = **시간 영점 조절 회고.** "방금 15분, 체감 어땠나요?" 3버튼(순식간/딱 15분/너무 길게). 타이머는 15분에 반드시 끊김(자동 연장 없음).
**대안:** '5분 더 연장' / '휴식 권장' 단독.
**근거:** 무작정 연장은 작업량은 늘려도 "이게 15분이구나"를 못 꽂아 CBT 훈련 목적 미달. 완료 여부와 무관한 별개 축이라 성공/실패 양쪽 모두 존재. + 예측 적중 보너스([D-16](#d-16))는 침묵 규칙(적중 시에만 노출).

## D-12

**결정:** 집중 화면 = 매우 단순(남은 시간 텍스트만). 시각 장치(모래시계·자라나는 나무·원형 진행 게이지) ❌(아이디어 킵).
**근거:** 화려한 애니메이션은 도파민 민감한 ADHD에게 '또 다른 볼거리' → 딴짓. 지루한 텍스트 화면은 폰을 뒤집고 현실 과제에 몰입하게 유도(물리적 단절). 구현 복잡도↓. 종료는 **부드러운 소리/진동**(요란한 알람·빨간 화면·"시간 종료!" ❌). 예외: 딴생각 포착 입력 1칸 허용.

## D-13

**결정:** 인지 과부하에서도 대처를 활성화하는 짧은 만트라를 앱 곳곳에 노출("막막하면, 조각이 너무 큰 거예요"). 반복 실패 패턴에 외부 이름을 붙여 자기비난과 분리(실패 외재화).
**근거:** 부적응적 자동사고를 반박·교정하는 CBT 레이어(장기 효과 핵심). **저작권:** 원문 만트라·글리치 명칭(예: "Time Bandit") 사용 금지 — 원리만 차용, 문구·명칭은 컴페이스 자체 창작([부록A](#부록a-법적ip)). 실패 외재화(패턴 네이밍)는 데이터 필요 → 후순위.

## D-14

**결정:** 자잘한 일은 낱개로 안 띄우고 '가벼운 15분 바구니' 블록에 묶음. **단, MVP 보류(post-MVP).**
**근거:** 바구니는 *시동 걸기 지연*이 아니라 *리스트 과부하*를 푸는 별개 페인포인트이며 핵심 영점 조절 루프가 그대로 안 맞음(별도 모드). MVP는 단일과제 훈련 루프를 먼저 깨끗이 검증. 니즈가 실제로 오르면 Phase 2 재개.

## D-15

**결정:** 블록 길이 = **15분 단일 고정.** 사용자가 길이를 고르지 않음.
**대안:** 15/30분 복수 고정(보류·킵) / 자유 입력(영구 폐기).
**근거:** "몇 분?"이라는 열린 판단은 시동 순간에 결정 피로/마비를 정확히 재발. 30분 옵션은 완벽주의자가 큰 덩어리를 골라 시동 허들을 _높이는_ 함정([D-13](#d-13) "조각이 너무 크다"와 충돌). 15분 고정은 연장 차단([D-11](#d-11))과 한 몸.
**재검토 트리거:** 영점 조절에서 `[순식간이었다]`가 특정 사용자에게 반복 관측되면, 길이는 사용자가 처음부터 고르는 게 아니라 **시간 감각이 길러진 결과로** 앱이 제안(2단계 해금)하는 것을 재논의.

## D-16

**결정:** 사전 예측 단위 = **'분'이 아니라 '블록 안/밖'.** "이번 15분에 끝날까?" 2지선다.
**근거:** 15분 고정([D-15](#d-15))으로 예측할 '분'이 없어짐 + ADHD 군집 지각(시간을 선형'분'이 아닌 덩어리'블록'으로 지각)에 부합. 예측값은 종료 후 완료 여부와 대조 → 예측 적중 보너스. **침묵 규칙:** 적중 시에만 보너스, 빗나감은 무표시(부정 문구 금지). **게이밍 주의:** 보너스는 작게, 톤은 "맞히기 게임" 아니라 "시간 감각이 자라고 있어요". 다중 블록 추정은 후순위.

## D-17

**결정:** 데이터 4층(북극성·현재 위치·큰 과제·행동 블록). 화면엔 3층만 노출, **큰 과제 층은 숨김**(맥락 표기만). **큰 과제도 One Task**(동시 1개, 다 쪼개 쓰기 전 새 과제 ❌).
**근거:** 목록/트리로 펼치면 우선순위 마비 재발. 큰 과제 층까지 One Task로 묶어 정체성(list 회피)을 끝까지 보존. 예외: 쪼개기 화면의 블록 목록은 _쪼개는 동안에만_ 존재, 시작 시 닫힘(상주 목록 ❌).

## D-18

**결정:** 방전 모드 = 진짜 One Task 블록을 **그대로 유지**, 승리 조건만 **"타이머만 켜도 오늘의 승리"**로 완화.
**대안(폐기):** 과제를 "책상에 앉기" 등 캔 액션으로 **치환** — K 인터뷰 오독으로 판명.
**근거:** 배터리 5%인 날 "15분 블록 시작"조차 천장([D-03](#d-03) 가정①). K의 병은 "진짜 과제를 못 봄"이 아니라 _생각→행동의 다리가 끊긴 것_ → "완성해야 한다" 제약 제거 + 작은 동작 트리거가 답("타이머만 켜면 승리"가 곧 그 트리거).
**경계:** 블록 *길이 선택지*를 다시 여는 것과 혼동 금지([D-15](#d-15)와 무충돌) — 이건 *승리 조건 완화*지 길이 옵션이 아니다. 에너지 바 죄책감 없음.
**→ SPEC:** 발동 = **상시 노출 저마찰 링크**("오늘은 가볍게 갈까요"), 회고 전체 스킵, 진짜 과제 보존 → `SPEC.md` 방전 모드 §.

## D-19

**결정:** 북극성 단수 강요 금지. _의무·열망_ 둘 다 / "아직 모르겠어요"까지 허용, **순위 강요 없음**, 건너뛰기 가능.
**대안:** 북극성을 하나만 입력하도록 강요.
**근거:** K의 책상 마비는 _두 북극성의 충돌_([D-03](#d-03) 가정②). 앱은 그 충돌을 **해결하지 않아도 되는 유일한 장소**여야 한다. 사용자는 매일 "지금 가장 만만한 한 조각"만 고름([D-05](#d-05)). 순위 판단 요구는 마비 재발.
**→ SPEC:** 온보딩에서 북극성을 강요하지 않고 뒤로 밀림/건너뛰기 → `SPEC.md` 온보딩·북극성 §.

## D-20

**결정:** K의 도파민 3다리(즉각·표현·사회적) 중 **두 다리는 되돌려주고 한 다리는 안 짓는다.**

- ✅ **즉각성 복원:** 보상은 15분 종료 그 순간 즉시(주간 집계 ❌). 강도가 아니라 **지연**을 줄인다.
- ✅ **표현성 복원:** 딴생각 포착 + 회고 3버튼을 "관객 없이도 쏟아내는 창구"로 재정의(새 UI 아님).
- ❌ **사회적 관객 배제(anti-goal):** 응답형 독자/응원단/도파민 슬롯머신을 의도적으로 짓지 않음.
  **근거:** 사회적 다리에 의존하도록 배선된 것 자체가 병의 일부. 목표는 "더 센 도파민"이 아니라 "응답 없어도 그 15분은 값지다"의 몸 재배선(과잉정당화 효과 방어 = 자립). 에너지 바의 고독함은 버그가 아니라 **의도**. 즉시성은 집중 화면 장식이 아니라 *전환(회고·보상) 순간*에만([D-12](#d-12)와 정합).

## D-21

**결정:** 앱은 **잔소리하는 부모의 냄새**를 절대 풍기지 않는다. 침묵 규칙을 '부재'까지 확장.
**금지:** 재접속 유도 알림("3일째 안 오셨어요"), 스트릭 깨짐 처벌·표시, 검사형 "다 했어요?".
**근거:** K는 실행 직전 외부 마찰이 끼면 "귀찮으니 다 놓자"로 실행의 끈을 자름(반발심). 며칠 안 와도 침묵, 돌아오면 "다시 와줘서 반가워요"만. 밤/아침 자책 시점은 앱을 열면(pull, push ❌) "오늘 15분이라도 켠 당신"으로 리프레임 — 재정비된 북극성이 실제 화면이 되는 지점.

## D-22

**결정:** 온보딩 = 공감과 위로형(따뜻한 어른 톤). 3화면(공감→면죄부→안심).

## D-26

**결정:** MVP 플랫폼 = **설치형 PWA(웹).** 배포·설치 모델은 여기서 확정하며, **프레임워크·상태관리·저장 라이브러리 등 기술 스택 세부는 [`TECH-SPEC.md`](TECH-SPEC.md)가 소유**(React+Vite+TS·Zustand·IndexedDB·Vercel, 확정). 계정·백엔드·푸시 없음(local-first, 단일 기기). 저장 접근은 **얇은 저장 모듈 하나로 격리**(과설계 어댑터 프레임워크 ❌ — 접점이 적어 YAGNI).
**대안:** 네이티브(RN/Flutter) — 견고한 로컬 저장·안정 햅틱이나 스토어 설치 마찰이 §4 마찰 최소화 정면 위반(획득 단계)·n=1 K-튜닝 반복 지연·안 쓸 푸시 인프라. / 처음부터 Capacitor 래핑 — MVP에 빌드 복잡도·스토어 마찰 선지불(YAGNI).
**근거:** ① **백그라운드 실행 불요** — SPEC §6 "포그라운드에서만 흐름"을 P13에서 타임스탬프 카운팅으로 개정해 웹 최대 약점 제거. ② **푸시 불요** — pull-only(§4·R3, [D-21](#d-21))라 네이티브 핵심 이점이 무의미. ③ **마찰 최소화 = 제품 원칙** — K(배터리 5%·반발심)에게 "URL 열면 15분"이 스토어·계정보다 낮은 진입 마찰(§4의 물리적 형태). 설치 없음은 징검다리가 아니라 **정체성** — 네이티브 없이 PWA가 최종 형태여도 됨. ④ **반복 속도** — 단일 코드·즉시 배포로 K 정성 검증(§0) 최속 회전.
**K=Android 정합:** iOS 한정 리스크(저장 축출·Vibration 미지원)가 Android에서 소멸 — `storage.persist()` 영속 + Vibration/Wake Lock 웹 지원. iOS는 폐기 아니라 **유예된 옵션** — 진입 시에만 소환, 경로는 **Capacitor 래핑**(전면 재작성 ❌).
**경계 — 데이터 이식성(§0 직결):** 브라우저 IndexedDB ↔ Capacitor WebView 저장소는 다른 샌드박스라 래핑 시 자동 이전 ❌. "자라나는 증거"([VISION §1](VISION.md)) 유실은 치명. **MVP는 위험 사소**(에너지 매일 리셋·기록페이지 없음 → 날짜 간 누적 없음). 단 **post-MVP "증거 앨범"은 처음부터 export 가능하게 설계 필수**(데이터 쌓인 뒤 소급은 비쌈).
**P13 정합:** 물리적 단절([D-12](#d-12)) ↔ 포그라운드 정지(§6) 충돌 = 타임스탬프 카운팅으로 해소 → [SCREEN-FLOW P13](SCREEN-FLOW.md#4-논리-이슈-대장-전부-해소).
**→ TECH-SPEC:** 저장 경계 인터페이스·데이터 모델·프레임워크 선정은 [`TECH-SPEC.md`](TECH-SPEC.md)가 소유(확정). 암호화([부록A](#부록a-법적ip) "의료 준하는 암호화" 권고)는 MVP 보류 — 근거·리스크는 [TECH-SPEC §10](TECH-SPEC.md#10-암호화--mvp-보류-결정-및-리스크), 리스크 등재는 `SPEC.md` R5. **→ ROUTES:** URL/딥링크(방전 상시 링크 = 인앱 라우트).

## D-27

**결정:** 화면·컴포넌트 조립 규약(`DESIGN-SYSTEM.md`)에 Apple HIG(Human Interface Guidelines)의 **원리**(Clarity/Deference/Depth 3원칙·8pt 그리드 여백 리듬·타이포 위계·엘리베이션 서열·모션 절제)만 선택적으로 차용한다. **시각 정체성(SF Pro/SF Symbols/시스템 블루/iOS 내비게이션·탭바 크롬/iOS 고유 제스처)은 명시적으로 거부한다.**
**대안:** Material Design 3 원리 차용 / 자체 원리 처음부터 창안 / HIG 전체(원리+시각 정체성) 도입.
**근거:** ① `CLAUDE.md §8` "기획은 논문에서(What), 구현은 백지에서(How)" 원칙을 시각 시스템에도 동형 적용 — HIG는 업계에서 가장 성숙한 "위계·여백·깊이·모션" 원리 레퍼런스지만 원문 시각 언어(SF 서체·시스템 블루)를 그대로 이식하면 애플 앱처럼 보여 브랜드 정체성(`DB-01` 따뜻한 종이+테라코타) 정면 위반. ② K의 실기기 = Android([D-26](#d-26)) — iOS 시각 문법(탭바·내비게이션 크롬·제스처)을 이식하면 실제 사용 플랫폼과 불일치. ③ Material Design은 K=Android와는 맞지만 컴페이스가 이미 확정한 웜페이퍼·테라코타·명조 제목(`DB-01`~`DB-03`)과 Material의 시각 언어(피셔블 액션 버튼·잉크 리플·컬러 시스템)가 충돌해 오히려 재작업을 유발. ④ 원리만 차용하면 플랫폼 중립을 유지하면서(`TECH-SPEC.md` 스택과 무관) "왜 이 간격/왜 이 깊이 서열인가"의 근거를 업계 표준에 정박시킬 수 있다.
**경계:** 차용 = Clarity/Deference/Depth 3원칙, 8pt 그리드, 타이포 위계 개념, 엘리베이션 서열, 모션 목적성·절제. 거부 = SF Pro/SF Symbols/시스템 블루/iOS 내비게이션·탭바 크롬/iOS 제스처, HIG 원문 도식·명칭.
**→ SPEC/DESIGN-TOKENS:** 구체 적용은 [`DESIGN-SYSTEM.md`](DESIGN-SYSTEM.md)(여백 리듬·타이포 위계·깊이 서열·모션표) 참조. 토큰 값 자체는 `DESIGN-TOKENS.md` 불변(이 결정은 배치 규칙만 추가).
