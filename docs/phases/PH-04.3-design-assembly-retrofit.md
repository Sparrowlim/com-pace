# PH-04.3 — 디자인 조립 규약 소급 적용 (Design Assembly Retrofit)

> **의존:** [PH-04.2](PH-04.2-design-assembly-spec.md)
> **SSOT:** `docs/DESIGN-SYSTEM.md`(PH-04.2 산출물) · [phases/README.md §0](README.md#0-전역-규칙) 소급 감사 절차(읽기 전용 서브에이전트 병렬 패턴, PH-05/06 소급 감사 선례)
> **전역 규칙:** [phases/README.md §0](README.md#0-전역-규칙) 참조.

## Goal

PH-04.2가 만든 조립 규약을 나머지 UI 프리미티브 5종(`Button`·`Chip`·`EnergyBar`·`OptionRow`·`BottomSheet`)과 9개 페이지에 소급 대조하고, stylelint로 간격/색 하드코딩을 기계 차단하는 상태.

## 상세화 시점

PH-04.2 완료 후 착수 직전 세션에서 30~50개 체크리스트로 상세화한다(`phases/README.md §0` "착수 직전 위상만 상세화" 원칙 — 먼 미래 위상 과설계 방지).

## 예상 범위 (헤더만 — 착수 전 확정 아님)

- 6종 컴포넌트 + 9개 페이지 소급 감사(읽기 전용 서브에이전트 병렬, `phases/README.md §0` 절차 재사용)
- 편차 CRITICAL/HIGH/MEDIUM/LOW 분류. CRITICAL(가드레일 위반)은 즉시 수정, 그 외는 해당 PH 파일(PH-05~09) "구현 중 발견"에 기록 — **원 위상 완료 상태는 되돌리지 않음**(PH-05→PH-05.1 선례와 동일 처리 방식)
- stylelint 도입(`declaration-property-value-disallowed-list` 등으로 raw px 간격·raw hex 색 하드코딩 차단, `web/hooks.md` 권장 항목 반영)
- `evidence.fill`·`action` 등 신성불가침 토큰 재확인(회귀 0건 보장)

## Positive Non-Goals (헤더만, 상세화 시 재확인)

- 토큰 값 재조정 없음(`tokens/design-tokens.json` 불변)
- 신규 컴포넌트 신설 없음 — 기존 6종의 배치·클래스 내부만 조정
- 사용자 행동/화면 전이 변경 없음(순수 시각 리팩터링)
