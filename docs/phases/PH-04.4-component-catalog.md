# PH-04.4 — 컴포넌트 카탈로그 v2 (SCREEN-FLOW 전체 커버리지)

> **의존:** [PH-04.3](PH-04.3-design-assembly-retrofit.md)(완료, 6종 프리미티브 소급 감사) · [PH-05.1](PH-05.1-core-loop-remediation.md)·[PH-06.1](PH-06.1-carryover-requeue.md)·[PH-07](PH-07-onboarding.md)·[PH-08](PH-08-discharge.md)·[PH-09](PH-09-settings-northstar.md)(전부 완료 — 이 위상이 소급 감사·리팩터하는 산출물)
> **SSOT:** [`DESIGN-SYSTEM.md`](../DESIGN-SYSTEM.md)(조립 규약, §5·§6 신설 대상) · [`DESIGN-TOKENS.md` §3 결정#6·§5](../DESIGN-TOKENS.md#3-semantic-토큰-화면은-여기부터-참조)(가드레일 바인딩) · [`SCREEN-FLOW.md` §1](../SCREEN-FLOW.md#1-화면상태-인벤토리)(화면 인벤토리 15개) · [`CLAUDE.md` §1·§2·§6](../../CLAUDE.md)(정체성 경계·불변 규칙·신규 기능 체크리스트) · [`common/coding-style.md`](../../../../.claude/rules/ecc/common/coding-style.md)(DRY)

## Goal

`DESIGN-SYSTEM.md`에 §5(컴포넌트 카탈로그)·§6(조립 레시피)이 신설되어 SCREEN-FLOW §1의 15개 화면 전부가 "카탈로그에 있는 컴포넌트의 조합"만으로 설명 가능하고, 감사로 발견된 5개 미등재 컴포넌트(`TextInput`·`TimerDisplay`·`StateChip`·`BonusCard`·`NorthStarBadge`)가 `src/components`에 구현되어 기존 페이지의 ad-hoc CSS를 대체한 상태.

## 배경 — 어쩌다 이 위상이 필요해졌나

PH-04.3은 PH-04가 만든 **6종 프리미티브**(`Button`·`Chip`·`TaskCard`·`EnergyBar`·`OptionRow`·`BottomSheet`)만 소급 감사했다. 그런데 그 이후 완료된 PH-05.1/06.1/07/08/09가 각자 페이지 CSS 모듈로 **회고 인정칩·보너스 카드·타이머 디스플레이·텍스트 입력(3곳 중복)·북극성 배지**를 그때그때 재발명했고, 어느 것도 `DESIGN-SYSTEM.md`에 등재되지 않은 채 각 위상이 "완료" 처리됐다. `phases/README.md §0`의 SPEC 커버리지 게이트는 "SPEC에 있는데 코드가 없는 것"만 잡고, PH-04.3의 소급 감사는 "이미 있는 6종의 배치"만 봤다 — **"코드에는 있는데 카탈로그에 없는 UI 패턴"을 잡는 절차가 없었다.** 이 위상이 그 절차(컴포넌트 카탈로그 게이트, `phases/README.md §0` 신설)와 이번 세대의 소급분을 함께 처리한다.

## Phase 1 — UX 근거 수립 (컴포넌트별 워크시트)

> **이 절이 이 위상의 핵심이다.** 각 신규 컴포넌트는 "이 토큰을 쓴다"가 아니라 **"왜 이 형태이고 왜 다른 형태가 아닌가"**를 페르소나 K(CLAUDE.md 설계 기준선 — 약물 중단·우울 동반·배터리 5%·목표가 의무↔열망으로 찢김)의 구체적 마찰 시나리오에 대고 검증한다. 기각한 대안을 남기는 이유: "왜 더 화려한/기능적인 버전을 안 만들었나"가 나중에 재질문되지 않도록.

### 1-6. 조립 레시피 UX 메모 (신규 컴포넌트 없이 기존 조합으로 충분한 것)

- **`CaptureModal`(5-A 딴생각)** = `BottomSheet` + `TextInput multiline` + `Button`. K 시나리오: 타이머가 안 멈추는 상태에서 입력하므로 인지 부하가 이미 높다 — 새 primitive를 만들지 않고 기존 `BottomSheet`(elevation.sheet)를 그대로 써서 "이건 또 다른 종류의 화면"이라는 학습 비용을 추가하지 않는다.
- **`PauseOverlay`(5-B 일시정지)** = `BottomSheet` + `Button`×2(재개/그만하기). "그만하기"는 죄책감 어휘 없는 중립 표현 유지(이미 구현됨, 카탈로그 등재만 필요). 버튼 2개 모두 44×44px 이상 — 배터리 5%·손 떨림 상태의 오탭 방지가 곧 마찰 최소화(§4).
- **`DischargeLink`(방전 진입)** = `Button variant="secondary"` 재사용. 신규 "QuietLink" primitive를 검토했으나 기각 — secondary 버튼이 이미 `action`(테라코타 elevation.cta) 없이 아웃라인만 쓰므로 §5-3 위반 소지가 없고, 새 컴포넌트를 만들면 오히려 "방전 링크는 특별하다"는 시각적 신호가 되어 §4 "고장 라벨 없이 초대 톤만"의 취지와 어긋난다.
