# 화면 구성 (Composition) — `CMP`

> 상위: [`../DESIGN-SYSTEM.md`](../DESIGN-SYSTEM.md) 허브 · 간격 크기: [`spacing.md`](spacing.md) · 화면 원형별 적용: [`decision-guide.md`](decision-guide.md)
> 이 문서는 **"요소를 화면 세로 어디에 두는가"**를 소유한다. [`spacing.md`](spacing.md)가 "얼마나 벌리는가(간격 크기)"라면, 이 문서는 "어디에 두는가(배치·무게중심)"다.
> **위계:** 아래 어떤 배치 규칙도 CLAUDE §2(담백)·§3(도파민 anti-goal)·§6-6(볼거리 금지)을 깨지 못한다. 특히 **CMP-6**을 먼저 읽는다.

## 왜 이 문서가 생겼나

화면이 "비어 보이거나 뭉쳐 보이는" 편차 — 콘텐츠가 상단에만 쌓이고 가운데가 목적 없이 비며 하단에 요소 하나가 외롭게 떠 있는 **바벨(barbell) 레이아웃** — 은 배치 규정의 공백에서 나온다. 세로 배치를 "버튼 y좌표 고정" 한 가지 관심사로만 다루면(구 §1-1), 그 부작용으로 정의되지 않은 공허가 생산된다. 이 문서는 앵커 규칙을 **폐기하지 않고 위로 한 겹 올려**, 공허를 담백한 여백으로 프레임하거나 좁히는 규정을 더한다.

## CMP-1. 세로 3존 (모든 화면 공통 골격)

모든 화면은 위→아래 세 존으로 조립한다.

| 존         | 내용                                         | 배치                                         |
| ---------- | -------------------------------------------- | -------------------------------------------- |
| **헤더존** | 북극성 배지·설정 진입·방전 종료 배너 등 크롬 | 화면 최상단 고정                             |
| **초점존** | 그 화면의 대표 콘텐츠(카드·질문·입력·타이머) | 헤더 바로 밑 밀착 ❌ — CMP-3 초점 밴드       |
| **앵커존** | 주 행동 버튼군 + 그날의 증거(`EnergyBar`)    | 뷰포트 하단 고정(`margin-top: auto`) — CMP-2 |

## CMP-2. 앵커 불변 (ADHD 공간 일관성)

주 행동 버튼군은 **앵커존(뷰포트 하단 고정)**에 둔다. 콘텐츠 길이와 무관하게 버튼 y좌표가 흔들리지 않아야 한다 — 실행 기능 저하 상태(페르소나 K, CLAUDE §1·§4)에서 "버튼이 이번엔 어디 있지"를 매번 다시 찾는 것 자체가 마찰이기 때문이다.

```css
.page {
  display: flex;
  flex-direction: column;
  min-height: 100svh;
  padding: var(--space-6);
}
.actions {
  margin-top: auto; /* 앵커존 — 마지막 자식 */
}
```

- **세로 중앙정렬 금지(폐기됨).** 콘텐츠 길이가 화면마다 다르면 중앙정렬된 버튼의 실제 위치도 흔들려 이 원칙과 충돌한다.
- **두 그룹 규칙:**
  - **카드 없는 화면** — 콘텐츠는 상단에서 시작, 행동 버튼군은 앵커존 하단 고정. 적용: `NorthStarPage`(1-B)·`PredictPage`(4)·`SettingsPage`(9)·`RestPage`(6-A)·`DischargeEntryPage`·`RetroPage`(6/6′/7/7′).
  - **`TaskCard`로 감싸는 화면** — 카드 자체가 시각적 앵커이므로 상단에서 시작(중앙정렬도, 하단 고정도 아님, 카드는 항상 헤더 바로 아래). 적용: `OnboardingPage`·`SplitPage`·`DashboardPage`(카드 상태)·`DischargeDashboardPage`.
- **의도적 편차(유지):** 두 그룹의 주 행동 버튼이 서로 다른 절대 위치에 있는 것은 앱 전체 단일 기준으론 편차이나, 통일하려면 `TaskCard` 재구조화가 필요하고 이는 검증된 인터랙션을 건드리는 더 큰 변경이라 **의도적으로 유지**한다(`zero-dashboard.spec.ts`가 zero 대시보드의 `[data-task-card]` 개수 0을 하드 검증 — "카드 없음"은 결함이 아니라 온보딩 1-A/2z/3-A와 컴포넌트를 겸용하는 명시적 설계).

## CMP-3. 초점 밴드 (focal band)

초점 콘텐츠(대표 카드·질문·입력·타이머)를 **헤더 바로 밑에 밀착시키지 않는다.** 헤더존과 초점존 사이에 의도적 상단 여백(섹션-간 계층 `space.7`~`space.8`)을 두어, 초점 콘텐츠가 "위에 처박힌" 느낌 대신 **"무대에 놓인" 느낌**이 되게 한다.

- 초점 콘텐츠의 무게중심은 **상단 1/3 ~ 광학 중앙** 사이에 둔다(뷰포트 최상단도, 정확한 기하 중앙도 아님).
- 이 상단 여백은 앵커(CMP-2)와 충돌하지 않는다 — 초점존 *안에서만*의 배치이고 앵커존은 여전히 하단 고정이다.

## CMP-4. 공백은 프레임되어야 한다 (void must be framed)

초점존과 앵커존 사이의 빈 공간은 **결함이 아니라 담백함의 여백일 수 있다.** 단 아래 두 조건을 모두 만족할 때만 여백으로 읽히고, 아니면 "고장"으로 읽힌다.

1. **초점 콘텐츠가 충분한 presence를 가진다** — 대표 카드는 옹색한 최소 크기가 아니라 생성적 내부 여백(외곽 패딩 `space.7`, 제목-본문-액션 사이 카드-내부 계층)으로 "의도된 무대"의 무게를 갖는다. 짧은 콘텐츠("123" 한 줄)여도 카드 자체는 담백하되 옹색하지 않게.
2. **앵커존이 비어 보이지 않는다** — CMP-5.

"작은 카드 + 보이지 않는 앵커"의 조합은 여백이 아니라 바벨 공허다.

## CMP-5. 앵커존 비가시 방지

앵커존의 요소가 전부 조건부라 사라지면(예: `EnergyBar`가 `filledCount=0`이라 캡션 없음 + `DischargeLink` 부재) 앵커가 텅 빈 것처럼 보인다.

- 앵커존에는 **항상 최소 하나의 가시 요소를 보장**하거나, 없으면 초점존이 그 세로 공간을 자연스럽게 흡수하도록 재구성한다.
- 해법은 **재배치(⭕)**이지 장식 추가(❌)가 아니다. 특히 `EnergyBar`의 빈 상태를 미리 그린 빈 칸으로 메우는 것은 영구 금지(`../DESIGN-TOKENS.md §6` anti-token `color.energy.empty` · SPEC §8 "0에서 자람").

## CMP-6. 채움 ≠ 장식 (하드 경계)

위 어떤 규칙도 **볼거리·틴트·일러스트·플레이스홀더 카드·마스코트로 공백을 메우는 것**을 정당화하지 않는다.

- CLAUDE §3(도파민 슬롯머신 anti-goal) · §6-6(집중 화면 볼거리 금지) · `../DESIGN-TOKENS.md §6`(anti-token) 위반.
- 공백은 오직 **비례(proportion) · presence · 재배치**로만 다룬다.
- 판정 질문: "이 요소가 없어도 화면이 자기 일을 하는가?" 그렇다면 그것은 장식이므로 넣지 않는다.
