# 판단 가이드 — 언제 무엇을 (Decision Guide) — `DG`

> 상위: [`../DESIGN-SYSTEM.md`](../DESIGN-SYSTEM.md) 허브 · 세로 배치: [`composition.md`](composition.md) · 프리미티브: [`components.md`](components.md) · 화면별 조합: [`recipes.md`](recipes.md)
> **이 문서의 역할:** 규약([`spacing`](spacing.md)·[`typography`](typography.md)·[`elevation`](elevation.md)·[`motion`](motion.md)·[`composition`](composition.md))이 _무엇을_ 고정한다면, 이 문서는 *어떤 상황이면 어떤 조립을 고르는가*를 준다. 새 화면을 짤 때 여기서 원형을 먼저 고른 뒤 규약을 적용한다.

## DG-1. 화면 원형 → 구성 패턴

먼저 만들 화면이 아래 6원형 중 무엇인지 정한다([`recipes.md`](recipes.md) "원형" 열과 1:1 대응).

| 원형         | 언제 이것인가                                        | 초점존 콘텐츠                             | 앵커존                         | 초점 밴드(CMP-3)                               |
| ------------ | ---------------------------------------------------- | ----------------------------------------- | ------------------------------ | ---------------------------------------------- |
| **카드형**   | "지금 이 한 조각"을 무대에 올릴 때(홈·온보딩·쪼개기) | `TaskCard`(대표 카드)                     | `Button`(주 CTA) + `EnergyBar` | 카드가 상단 밴드에 presence 있게(옹색 ❌)      |
| **입력형**   | 사용자가 한 줄 적을 때(아무거나 입력·북극성)         | `TextInput`(±질문 문구)                   | `Button`(다음/저장)            | 질문+입력을 상단 밴드에, 버튼은 앵커 하단      |
| **선택형**   | 닫힌 2~3지선다(사전예측·휴식·방전진입)               | `OptionRow`×N 또는 안내 문구              | `Button`(또는 선택이 곧 확정)  | 선택지를 상단 밴드에 모아, 하단은 앵커         |
| **타이머형** | 집중 15분(볼거리 금지 최강)                          | `TimerDisplay`(숫자+동사)                 | (없음 — 딴생각/정지는 시트로)  | 숫자를 광학 중앙에, 그 외 전부 제거            |
| **회고형**   | 15분 후 인정·영점조절(적중 시 가산)                  | `StateChip`+`OptionRow`×3(+`BonusCard`)   | `Button` + `EnergyBar`         | 인정칩을 상단, 증거바+버튼을 앵커              |
| **시트형**   | 흐름을 끊지 않는 보조 개입(딴생각·일시정지)          | `BottomSheet` 내부에 `TextInput`/`Button` | 시트 자체가 오버레이           | N/A — 시트는 3존 골격 밖(하단에서 밀고 올라옴) |
| **목록형**   | 설정 등 항목 나열                                    | `OptionRow`×N(토글)                       | `Button`(저장/닫기)            | 목록을 상단, 하단은 앵커                       |

## DG-2. "비어 보이는가"를 짤 때 자문하는 순서

새 화면(특히 콘텐츠가 짧은 화면)이 바벨 공허로 빠지지 않게 하는 체크 — [`composition.md`](composition.md) CMP-3~6의 적용 절차:

1. **초점 콘텐츠에 presence가 있는가?** 대표 카드가 옹색한 최소 크기면 외곽 패딩(`space.7`)·초점 밴드 상단 여백을 먼저 준다. (CMP-3·CMP-4-①)
2. **앵커존이 비어 보이지 않는가?** 하단의 유일 요소가 조건부로 사라지는가(`EnergyBar` at 0 등)? 그렇다면 가시 요소를 보장하거나 초점존이 흡수하게 재배치한다. (CMP-5)
3. **남은 공백이 여전히 크면 — 그대로 둔다.** 프레임된 여백은 담백함의 자산이다. (CMP-4)
4. **절대 하지 않는 것:** 볼거리·틴트·일러스트·플레이스홀더로 메우기. (CMP-6 하드)

## DG-3. "새 컴포넌트가 필요한가"를 물을 때

기존 11종([`components.md`](components.md))으로 설명이 안 될 때만 신설한다. 신설 전 반드시:

1. **기존 조합으로 충분한가?** [`recipes.md`](recipes.md) RC-2의 기각 사례(딴생각→`BottomSheet` 재사용, 휴식→방전진입 패턴 재사용)를 먼저 본다.
2. **볼거리를 더하려는가?** 그렇다면 CLAUDE §6-6·CMP-6 위반 — 만들지 않는다.
3. **판단/선택을 새로 요구하는가?** 그렇다면 CLAUDE §2 결정 피로 차단 위반 — 재설계.

## DG-4. "이 색/발광을 써도 되는가"를 물을 때

- **테라코타 CTA(`action`·`elevation.cta`)?** → "즉시성의 순간"(타이머 시작·블록 확정·적중)인가? 아니면 `variant="secondary"` 아웃라인. (EL-3 · `../DESIGN-TOKENS.md §5-3`)
- **회고 그린/퍼플(`state.*`)?** → 회고 인정칩 안인가? 밖으로 새면 성적표 프레임 위반. (`../DESIGN-TOKENS.md §3 결정#6`)
- **에너지 칸 색?** → 언제나 `evidence.fill` 하나. 완료/미완료/방전 분기 절대 금지. (`../DESIGN-TOKENS.md §5-1`)
