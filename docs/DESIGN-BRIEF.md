# 컴페이스 — 디자인 브리프 (Design Brief)

> **역할:** 시각·인터랙션·톤의 **방향과 원칙(HOW-it-feels).** 논리(SPEC)와 와이어 재작업 사이의 **다리.** 미적 결정을 매 화면 즉흥 재litigate하지 않도록 가드레일을 시각 언어로 번역한다.
> **정본 경계:** 확정 스코프·규칙 = `SPEC.md` · 화면 전이 = `SCREEN-FLOW.md` · 의도 = `VISION.md` · 결정 근거 = `DECISIONS.md` · 불변 규칙 = `/CLAUDE.md`.
> **위계:** 본 브리프는 불변 규칙(§2)·SPEC을 **깨지 못한다.** 브리프의 미적 선택이 SPEC과 어긋나면 SPEC이 최신이다. 브리프는 "어떻게 보이고 느껴지는가"만 소유한다.
> **성격:** 이것은 *디자인 시스템*이 아니라 *브리프*다 — 방향·원칙·규칙까지. 최종 토큰값·컴포넌트 스펙·최종 카피는 §7 유예 목록으로 명시적으로 미룬다.

---

## 0. 한 줄 요약 (이 문장이 흔들리면 디자인이 산으로 간다)

> **"위로(따뜻한 종이)를 기본으로 깔되, 판단·행동의 순간엔 화면을 비워 규율한다(고요한 미니멀)."**

자책하는 삶에 대한 **위로**와 **행동교정**을 동시에. 위로 = 배경·서체·톤의 온기. 행동교정 = 집중·회고·보상 순간의 담백한 절제. 모든 디자인 결정(D-01~D-06)은 이 이중성의 파생이다.

| --------- | ------------------- | ---------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------- |

| **DB-02** | 색 팔레트           | **흙빛 중립 + 테라코타(적중/즉시성 순간만) + 방전 전용 차분톤** · 처벌색 없음(하드) · 액센트=즉시성 순간만(하드) | §8 무처벌 · §3 즉시성 · §5 방전 차분                                    |

## 2. 스타일 방향 (DB-01)

**무드:** 따뜻한 어른이 종이에 손으로 적어주는 느낌. 크림 배경·둥근 모서리·부드러운 그림자.

**규율 규칙:** 사용자가 **판단하거나 행동하는 순간(집중·회고·보상)엔 화면을 더 비운다.** 위로는 배경에, 규율은 행동의 순간에.

- ✅ 온보딩·북극성·대시보드: 따뜻함 강하게
- ✅ 집중·회고·보상: 고요·담백 강하게
- ❌ 귀여움 과잉(물렁), 차가운 미니멀(무심), 화려한 볼거리 — 셋 다 이중성 위반

## 3. 디자인 토큰 (규칙 우선 · 값은 잠정)

> **중요:** 아래 hex/px 값은 **와이어에서 추출한 잠정값**이다. 구현·와이어 재작업 때 미세조정한다(§7). 여기서 확정하는 것은 **"각 토큰이 무슨 뜻이고 어디에만 쓰는가"** 이다.

### 3-1. 색 (DB-02)

| 토큰                  | 뜻 / 사용처                                    | 잠정값 (와이어 추출)                                   |
| --------------------- | ---------------------------------------------- | ------------------------------------------------------ |
| `color-surface`       | 앱 배경(종이)                                  | `#E7DECF` / 카드 `#F6F1E6`                             |
| `color-text`          | 본문 진한 글자                                 | `#4A423B`                                              |
| `color-text-muted`    | 보조 글자                                      | `#8C8070` → 더 흐림 `#A89A82`                          |
| `color-accent`        | **테라코타 — 적중/즉시성의 순간에만**          | `#E79155` / 진한 `#C9743C`                             |
| `color-accent-soft`   | 액센트 배경 칩                                 | `#F3DDC8`                                              |
| `color-energy-filled` | 에너지 채운 칸                                 | `#E79B62`                                              |
| `color-energy-empty`  | (자라나는 바에선 미리 안 그림 — 경계선 용도만) | `#E4DAC7`                                              |
| `color-calm`          | **방전 모드 전용 차분톤**                      | **미정 — 제안: 흙빛에서 채도 뺀 그레이지/뮤트 세이지** |

**하드 규칙 (색):**

1. **처벌색 없음** — 실패/미완료를 뜻하는 빨강·경고색은 팔레트에 **존재하지 않는다.** (§8)
2. **액센트 = 즉시성의 순간에만** — 버튼마다 도배 금지. 테라코타가 켜질 때 특별해야 한다. (§3)
3. **방전 차분톤은 방전 모드에서만** — 다른 곳 전용.

### 3-2. 타이포그래피 (DB-03)

| 역할                            | 폰트                                   | 비고                                  |
| ------------------------------- | -------------------------------------- | ------------------------------------- |
| 제목·환영·만트라(는 아님, §5-2) | **따뜻한 명조** (후보: `Gowun Batang`) | 손편지·책의 온기. 면적 작아 부담 적음 |
| 본문·버튼·UI                    | **고운 산세리프** (`Gowun Dodum`)      | 기능부는 깨끗·담백                    |

- 폰트 **최대 2종** (성능). 구현 시 `font-display: swap` + 서브셋.
- Jua(기존 제목 폰트) **폐기** — "어린이 앱" 인상 = §4 반발심 위험.

### 3-3. 형태·모션 (DB-04, DB-06)

- radius(잠정): 카드 34px · 버튼 18px · 아이콘 14px
- **집중 화면 = 카운트다운 + 딴생각 1칸 외 완전 정지** (볼거리 0, §6)
- **유일한 의도적 모션 = 보상 점등** — 수백 ms, 부드럽게. 반짝임·튀어오름·파티클 **금지**(§3 anti-goal)
- **`prefers-reduced-motion` 존중** — 켜지면 점등도 즉시 상태변화로 대체
- 애니메이션 속성 = **transform/opacity만**
- 화면 전환("대시보드 낙하" 등) = 부드럽고 빠르게, 연출 금지
- **방전 모드 = 더 잔잔하게**

## 4. 톤 & 목소리 (DB-05)

**목소리의 주인공:** **따뜻한 큰 어른** — 다 겪어본, 재촉하지 않는 손윗사람. 짧게 말하고, 검사하지 않고, 결과가 아니라 **네가 앉았다는 사실**을 조용히 알아봐 준다.

**발화량:** 핵심 순간(환영·면죄부·보상·돌아옴)에만 짧고 따뜻한 한두 문장. 그 외엔 침묵.

### 4-1. 하드 카피 규칙

1. **과정 칭찬만, 결과 칭찬 금지** — "완성했네요"(❌) / "15분 앉았네요"(✅)
2. **부정·검사·처벌 문구 금지** — "못 했네요" · "다 했어요?" · "며칠째 안 오셨어요" 전부 ❌ (침묵)
3. **부재도 침묵, 돌아오면 "다시 와줘서 반가워요"만** (§4)
4. **적중일 때만 조용히 축하**, 빗나감은 무표시 (§2)
5. **명령·재촉 금지**, 부드러운 존댓말, **초대 톤**(유도 아님)
6. **"치료" 클레임 금지** — 웰니스 포지션 (§8)
7. **자체 창작** — 논문 원문 문구·명칭 차용 금지 ([부록A](DECISIONS.md#부록a-법적ip))

### 4-2. 목소리 예시 (Before → After)

| 상황           | ❌ 흔한 앱             | ✅ 우리                                           |
| -------------- | ---------------------- | ------------------------------------------------- |
| 온보딩 첫 질문 | "가장 무거운 일은?"    | "지금 눈에 걸리는 거 아무거나, 사소해도 돼요."    |
| 15분 종료      | "🎉 미션 완료! 3/5!"   | "15분, 해냈어요. 한 칸 켜졌어요."                 |
| 미완료         | "실패했습니다 😢"      | _(성공과 동일 점등)_ "여기까지 온 것도 충분해요." |
| 오랜만 복귀    | "5일이나 안 오셨네요!" | "다시 와줘서 반가워요."                           |

## 5. 만트라 (DB-05 · CBT 인지 재구성 레이어)

**정체:** ADHD 인지 과부하의 순간, 자동 왜곡을 조용히 반박. **실패 책임을 *나* → *조각 크기/타이밍*으로 외재화**하는 CBT 무브. ([D-13](DECISIONS.md#d-13))

**배치 모델 = 저스트-인-타임 맥락 반박** (앰비언트 도배 ❌).

### 5-1. 트리거 → 반박 매핑

| 인지 왜곡이 켜지는 순간 | 어디 (기존 UI에 녹임)  | 반박 만트라 (자체 창작, 예시)                         |
| ----------------------- | ---------------------- | ----------------------------------------------------- |
| 시작 못 하고 막막함     | 쪼개기 화면 helper     | "막막하면, 조각이 큰 거예요. 더 작게 떼어도 돼요."    |
| 미완료 회고             | 미완료 화면 위로 한 줄 | "의지 문제가 아니라, 한 조각이 조금 컸던 것뿐이에요." |
| 딴생각이 자꾸 떠오름    | 딴생각 포착 모달       | "적어두면, 지금 것이 아니에요. 여기 맡겨둬요."        |
| 오래 안 오다 복귀       | 복귀 인사              | "다시 와줘서 반가워요."                               |
| 방전                    | 방전 대시보드          | "오늘은 시작한 것만으로 충분해요."                    |

### 5-2. 만트라 규칙

1. **맥락 전용** — 그 순간에만. 상시 노출·명언 카드·회전 배너 ❌.
2. **목소리로, 포스터로 ❌** — 별도 "만트라 스타일"(이탤릭 명언박스) 만들지 않음. 일반 카피와 **같은 톤·같은 서체.**
3. **외재화 방향 고정** — 책임을 *조각/타이밍*으로, 절대 *사람*으로 ❌.
4. **적중/성공엔 만트라 없음** — 반박이 필요한 곳(막힘·미완료·방전)에만.
5. **패턴 네이밍(실패 외재화)은 제외** — post-MVP ([D-13](DECISIONS.md#d-13) 후순위, SPEC §12).

## 6. 상태 시각 규약 (이 앱은 상태가 곧 제품)

- **에너지 바:** 0에서 자람 · 블록마다 +1 · 상한 없음 · 매일 리셋 · **미리 그린 빈 할당칸 ❌ · 분환산("N시간") ❌ · 날짜 비교 그래프 ❌** · 성공·미완료 **색·형태 동일** · 방금 켜진 칸만 일시 강조 후 동일화(DB-06). (SPEC §8)
- **회고 4조합:** 적중(6·7)만 **조용한** 보너스 / 빗나감(6′·7′)은 완료·미완료 화면과 **완전 동일(무표시)** / 미완료도 에너지 동일 상승. (SCREEN-FLOW §3-2)
- **방전:** 배경·크롬만 차분톤(`color-calm`)으로 무드 구분, **에너지 칸 자체는 정상과 동일** · 회고 스킵 · 시작 즉시 점등 + 따뜻한 한 줄. (SPEC §5)
- **대시보드:** 블록有/zero/방전 × 북극성有/無. zero = 담백한 증거 + 저마찰 새 과제, **성적표 프레임 ❌.** (SPEC §4)
- **집중 화면:** 카운트다운 + 딴생각 1칸 외 정지. (SPEC §4)
- **북극성:** 정적 의미 좌표만 · **진행 측정 UI(게이지·도달률·"어디쯤") ❌.** (SPEC §9)

### 6-1. 디자인 QA 체크리스트 (통과 기준)

- [ ] 부정·실패·처벌 색/문구가 어디에도 없는가 (§2)
- [ ] 성공과 미완료가 시각적으로 구분되지 않는가 (§8)
- [ ] 테라코타가 적중/즉시성 순간에만 쓰였는가 (DB-02)
- [ ] 집중 화면에 카운트다운 외 볼거리가 없는가 (§6)
- [ ] 만트라가 도배 아닌 맥락 순간에만, 목소리로 들어갔는가 (DB-05)
- [ ] 결과 칭찬이 아니라 과정 칭찬인가 (§4)
- [ ] reduced-motion에서 점등이 즉시 상태변화로 대체되는가 (DB-04)
- [ ] 템플릿(기본 카드 그리드·명언 포스터)처럼 보이지 않는가

## 8. 정합 액션 (브리프 확정에 따른 와이어/문서 수정)

1. 온보딩 카피 "가장 무거운 일"·"막막한 일부터" → **"아무거나·사소해도"** (SPEC §2)
2. 북극성 단수 "최종 목표" → **양가 두 좌표** (SPEC §7)
3. 제목 폰트 Jua → **명조** (DB-03)
4. 에너지 바 고정 5칸(3+빈2) → **0에서 자람** (SPEC §8)
5. 기록/통계 페이지 **삭제** (SPEC §12)
