> **정본 경계:** 의도·북극성·페르소나·안티골 = `VISION.md` · 결정 근거(WHY) = `DECISIONS.md` · 화면 전이 로직 = `SCREEN-FLOW.md` · 불변 규칙 = `/CLAUDE.md`.
> **위계:** 본 스펙이 `DECISIONS.md`·`CLAUDE.md`와 충돌하면 **이 스펙이 최신 확정본**이다(Grillme 세션 봉합 결과). 상위 문서는 이 스펙에 맞춰 정합.
> 유래: Grillme 워크플로우(모호성 제거·기능 범위 완전 확정, 문서↔와이어 모순 봉합).

## 0. 정체성 & 성공 지표 (위계 확정)

- **종착 목표 = `VISION.md` §1 북극성** (여기서 재기술하지 않는다). 성공 = "오늘 밤 자기를 조금 덜 미워하며 잠들었는가."
- **훈련은 수단, 종착이 목적:** 인지 훈련(시동·시간감각·인지 재구성)은 북극성 도달의 **방법**. 훈련 효능은 **동급 성공 지표가 아니다** → 내부 지표로만 관측(§10), 사용자 비노출.
- **이 제품이 훈련하는 것:** 시동 걸기 + 시간 감각(영점조절) + 인지 재구성. **우선순위 판단은 훈련 대상이 아니다**(의도적 보류, [DECISIONS D-05](DECISIONS.md#d-05)).

## 1. 페르소나 & 검증 체계

- **A(취준생):** 획득·마케팅용. 제작자 본인이 근접 → 자가 테스트 시 기능 과잉("뭐 하는 앱?") 위험이 있어 **설계 기준선으로 삼지 않는다.**
- **K(설계 기준선):** 29세·ADHD 약물 중단·우울 동반·배터리 5%·목표가 의무(공무원)↔열망(웹소설 작가)으로 찢김. **"5%의 K에게 통하면 모두에게 통한다."** (배경 = `VISION.md` §3, [DECISIONS D-03](DECISIONS.md#d-03))
- **검증:** K는 상주·상시 인터뷰 가능한 실재 인물 → §0("덜 미워하며 잠들었는가")은 **앱이 측정하지 않고 K 인터뷰로 앱 밖에서 정성·주기 관측**한다.

## 2. 온보딩 플로우 (확정)

```
면죄부 3화면 (공감→면죄부→안심)
  → "지금 눈에 걸리는 아무거나·사소해도 됨" 입력   ← 첫 질문 정정
  → 쪼개기 (수동 + 행동 동사 칩)
  → 대시보드 낙하 (블록 1개 노출)
  → [끝. 타이머·회고는 온보딩에 미포함]
```

- **면죄부 3화면 복원(확정):** 와이어에서 누락됐던 공감 3화면(고생 많았어요 / 게을러서 아니에요 / 15분 앉은 것만으로도 발전)을 **되살린다.** K의 RSD 첫 완충.
- **첫 질문 정정(확정):** "가장 무거운/막막한 일" → **"아무거나·사소해도 됨."** 무거운 걸 지목시키는 건 K의 회피 대상을 정면 노출하는 RSD 지뢰. 쪼개기는 _어떤_ 과제든 필요하지 _가장 무거운_ 과제가 필요치 않다. (이것이 "완충제"의 실체 — 새 장치가 아니라 질문 교체.)
- **북극성은 온보딩에서 강요하지 않는다** — 뒤로 밀리고 건너뛰기 가능.

## 3. 핵심 루프 (확정 · SSOT)

```
아무거나 한 조각 → 쪼개기(행동 동사) → 만만한 1개 자기선택
 → [사전 예측: 이번 15분에 끝날까? 2지선다]
 → 15분 고정 + [딴생각 포착]
 → 회고 ┬ 영점조절(체감 3버튼)
        ├ 예측 적중 보너스(적중 시에만, 침묵 규칙)
        └ 보상·이월(노력/완료 분리, 인지 재구성, 에너지 동일)
```

- **블록 1개 선정 = 사용자가 "만만함"으로 자기선택**(우선순위 판단 ❌ → 과부하 회피, 자기선택 유지 → 훈련 행위 보존). 앱이 대신 고르지 않는다.
- 상주 목록 없음([DECISIONS D-17](DECISIONS.md#d-17)) — 쪼개기 목록은 쪼개는 동안만, 시작 시 닫힘.

## 5. 방전 모드 (완전 확정)

- **정의:** 진짜 One Task 블록을 **그대로 유지**, 승리조건만 **"시작 = 오늘의 승리"**로 완화. (과제를 "책상에 앉기" 등 캔 액션으로 **치환하지 않는다** — 치환안은 K 인터뷰 오독으로 판명, [DECISIONS D-18](DECISIONS.md#d-18) 원안 "승리조건 완화"로 복귀.)
- **근거(K 인터뷰):** K의 병은 "진짜 과제를 못 봄"이 아니라 **생각→행동의 다리가 끊긴 것.** "완성해야 한다"는 제약 제거 + 작은 동작 트리거가 답. → **"타이머만 켜면 승리"가 곧 그 트리거**(둘은 같은 메커니즘).
- **진짜 과제:** 보존(파괴·소모 ❌). 다음날 그대로.
- **에너지 바:** 시작 순간 **즉시 점등**, 정상 블록과 **시각적으로 동일**(무처벌).
- **사전 예측:** **생략.** 회고가 전체 스킵되면 예측을 대조할 곳이 사라지므로 예측 단계 자체를 제거(마찰 최소화·결정 피로 차단). 방전 대시보드 → 곧장 집중.
- **회고:** **전체 스킵**(영점조절·예측보너스·이월) → 시작 점등 + 따뜻한 한 줄로 종료.
- **발동 주체:** **상시 노출 저마찰 링크**("오늘은 가볍게 갈까요") — 사용자를 '고장'으로 라벨하지 않음. (5% K가 "나 방전이야" 버튼을 누를 에너지가 없다는 모순 해소.)
- **설계 비용:** 별도 화면이되 새 콘텐츠 불필요 — 기존 블록의 **판정·카피만** 방전용 분기.

SCREEN-FLOW P4의 "다음날 그대로"는 이 경계로 읽는다.

## 7. 설정 페이지 (확정)

- **양가 목표(수정):** 단일 "최종 목표" → **의무·열망 두 좌표 나란히, 순위 없음**([DECISIONS D-19](DECISIONS.md#d-19), 대시보드 두 칩과 정합).
- **알림(유지·우수):** 기본 **꺼둠** + "재촉/확인 안 함 · pull-only" 카피 유지. opt-in 시 알림 **내용**이 "N일째 안 오셨어요" 류로 새지 않게 **카피 가드** 명시.
- **목표·라벨 "선택" + "둘 다 비우면 배지·라벨 사라지고 핵심 루프만"** — 유지.
- **대시보드 메뉴:** "📊 기록" 항목 제거, "⚙️ 설정"만.
- 계정 설정은 post-MVP.

## 8. 에너지 바 (확정)

- **자라나는 형태:** 0에서 시작, 블록마다 한 칸 점등. **미리 그린 빈 칸(할당량) 폐기** — 고정 5칸(3+빈2)은 "못 채웠네"를 유발해 무처벌 원칙 위반이었음.
- **분 환산 금지:** "N블록"만. "9시간 15분" 류 시간 누적 ❌(생산성 프레임, [DECISIONS D-10](DECISIONS.md#d-10)).
- **날짜 간 비교 그래프 금지:** 주간 막대·부재 요일 시각화 ❌(상대적 박탈감 + 부재까지 침묵).
- **성공/미완료 동일 점등**(무처벌). 완료/이어간 **분리 카운트 금지**(카피로 방어 불가).

## 9. 북극성 취급 (확정)

- **의미만 담당** — 진행 측정기 ❌. "얼마나 왔나"·도달률·책갈피식 "어디쯤" 표시 금지. "한 발짝" 증거는 **에너지 바**가 담당.
- **역할:** 정적 방향 좌표 + **K를 열망에 재접속**(재촉 아니라 재접속). 빈 북극성은 **"유도(압박)"가 아니라 "허용(초대)"** 톤으로만.
- **MVP 범위:** 상단 선택적 정적 라벨 + 양가 칩 + 건너뛰기. **블록↔방향 맥락 태그·진행 mechanic은 post-MVP.**

## 10. 내부 지표 (확정 · 사용자 비노출)

- **로깅:** 시동 성공률 · 예측 적중률 시계열 · 방전 진입 빈도 (전부). 루프 배출물만, 새 질문 ❌.
- **관측만, 최적화 타깃은 §0뿐**(Goodhart 방어).
- **어떤 사용자 노출·메시지·개입도 촉발 ❌.**
- 구현은 **post-MVP.** MVP 의무: ①"잘했/못했" 비노출 ②루프 배출물 로깅. **와이어 영향 0.**
